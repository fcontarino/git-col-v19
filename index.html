Areas de GIT

Working Directory

Es el area de trabajo, donde vamos a crear, editar, eliminar los diferentes archivos que formen parte de nuestro proyecto. Muy rara vez, estaremos utilizando la consola en esta instancia que basicamente estaria representada por la carpeta donde inicializamos el repositorio

Stagin Area (INDEX)

Es el area de control de cambios donde se realizan las capturas de codigo (snapshot) y basicamente donde indicamos a que archivos vamos a estar realizando seguimiento. Una vez agregados los cambios solamente debemos confirmarlos para agregarlos al repositorio.

Repository

Es el area donde se almacenan las confirmaciones de los cambios realizados en archivos de tipo BLOB (Binary Large Object File), se utilizan como puntos de referencia a los que podemos volver siempre y cuando indiquemos a git cual es el numero de HASH o posicion que queremos visualizar.
